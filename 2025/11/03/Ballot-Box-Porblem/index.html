Introduction

The Ballot problem is one of the most elegant results in combinatorics:
it counts how often one candidate can stay ahead in an election count.
Surprisingly, its logic also extends to games like matching pennies, where two players alternately win or lose coins — and we ask:

What is the probability that they are never even again?

We’ll start with the ballot theorem, then reinterpret it through lattice paths and end with the “matching pennies” variant.

The Ballot Box Problem

Imagine candidate A receives aaa votes and B receives bbb votes, with a>ba > ba>b.
The votes are counted in random order, and we ask:

What is the probability that A is always strictly ahead during the count?

Combinatorial setup & Gemoetric view

The Ballot theorem

The probability that A always leads is:

P(A always ahead)=a−ba+b.\boxed{P(\text{A always ahead}) = \frac{a-b}{a+b}}. P(A always ahead)=a+ba−b​​.

Extension: Matching Pennies

We now consider the matching pennies problem.

Two players, A and B, flip coins NNN times.
A gains +1+1+1 for each win and loses 111 for each loss.
Let SkS_kSk​ be the cumulative score after kkk rounds:

Sk=X1+X2+⋯+Xk,Xi∈{+1,−1}.S_k = X_1 + X_2 + \dots + X_k, \qquad X_i \in \{+1,-1\}. Sk​=X1​+X2​+⋯+Xk​,Xi​∈{+1,−1}.

We are interested in

PN=Pr⁡(Sk≠0 for all 1≤k≤N),P_N = \Pr(S_k \neq 0 \text{ for all } 1 \le k \le N), PN​=Pr(Sk​=0 for all 1≤k≤N),

the probability that they are never tied at any time during the match.

Setup

The parity of NNN matters because it determines whether a final tie (SN=0S_N=0SN​=0) is even possible.

  • Case 1: N=2nN = 2nN=2n (even) → possible to end tied (SN=0S_N=0SN​=0).
  • Case 2: N=2n+1N = 2n+1N=2n+1 (odd) → final score must be odd, cannot end tied.

We handle both cases separately.

Let kkk = number of A’s wins.
Then SN=2k−NS_N = 2k - NSN​=2k−N.
Each configuration with exactly kkk wins occurs with probability:

Pr⁡(A wins k)=(Nk)2N.\Pr(\text{A wins } k) = \frac{\binom{N}{k}}{2^N}. Pr(A wins k)=2N(kN​)​.

Applying Ballot Thoerem

For fixed kkk, the Ballot theorem gives the probability that the cumulative sum never hits 0:

Pr⁡(no tie∣k)=∣2k−N∣N,\Pr(\text{no tie} \mid k) = \frac{|2k - N|}{N}, Pr(no tie∣k)=N∣2k−N∣​,

since only when k=N/2k = N/2k=N/2 does the path ever return to the axis.

Thus the total probability is obtained by summing over all possible kkk:

PN=∑k=0NPr⁡(no tie∣k)⋅Pr⁡(A wins k)=1N2N∑k=0N∣2k−N∣(Nk).P_N = \sum_{k=0}^{N} \Pr(\text{no tie} \mid k) \cdot \Pr(\text{A wins } k) = \frac{1}{N 2^N} \sum_{k=0}^{N} |2k - N| \binom{N}{k}. PN​=k=0∑N​Pr(no tie∣k)⋅Pr(A wins k)=N2N1​k=0∑N​∣2k−N∣(kN​).

By symmetry, we can simplify each case.

Suming up using Algebra

Case A: N=2nN = 2nN=2n

When NNN is even, symmetry allows us to sum over k>nk > nk>n and double it:

P2n=22n 22n∑k=n+12n(2k−2n)(2nk)=1n 22n∑k=n+12n(k−n)(2nk).P_{2n} = \frac{2}{2n \, 2^{2n}} \sum_{k=n+1}^{2n} (2k - 2n) \binom{2n}{k} = \frac{1}{n\,2^{2n}} \sum_{k=n+1}^{2n} (k - n) \binom{2n}{k}. P2n​=2n22n2​k=n+1∑2n​(2k−2n)(k2n​)=n22n1​k=n+1∑2n​(k−n)(k2n​).

A known binomial identity simplifies the sum:

∑k=n+12n(k−n)(2nk)=n(2nn).\sum_{k=n+1}^{2n} (k-n) \binom{2n}{k} = n \binom{2n}{n}. k=n+1∑2n​(k−n)(k2n​)=n(n2n​).

Hence,

P2n=(2nn)22n.\boxed{ P_{2n} = \frac{\binom{2n}{n}}{2^{2n}}. } P2n​=22n(n2n​)​.​

Or equivalently, with N=2nN=2nN=2n,

P(no tie)=(NN/2)2N.\boxed{ P(\text{no tie}) = \frac{\binom{N}{N/2}}{2^{N}}. } P(no tie)=2N(N/2N​)​.​

Case B: N=2n+1N = 2n + 1N=2n+1

When NNN is odd, there is no middle tie term.
Similarly, we sum over k>nk > nk>n and double:

P2n+1=2(2n+1)22n+1∑k=n+12n+1(2k−(2n+1))(2n+1k).P_{2n+1} = \frac{2}{(2n+1) 2^{2n+1}} \sum_{k=n+1}^{2n+1} (2k - (2n+1)) \binom{2n+1}{k}. P2n+1​=(2n+1)22n+12​k=n+1∑2n+1​(2k−(2n+1))(k2n+1​).

Using another binomial identity:

∑k=n+12n+1(2k−(2n+1))(2n+1k)=(2n+1)(2nn).\sum_{k=n+1}^{2n+1} (2k - (2n+1)) \binom{2n+1}{k} = (2n+1)\binom{2n}{n}. k=n+1∑2n+1​(2k−(2n+1))(k2n+1​)=(2n+1)(n2n​).

Therefore,

P2n+1=(2nn)22n.\boxed{ P_{2n+1} = \frac{\binom{2n}{n}}{2^{2n}}. } P2n+1​=22n(n2n​)​.​

Rewriting in terms of N=2n+1N = 2n + 1N=2n+1,

P(no tie)=(N−1(N−1)/2)2N−1.\boxed{ P(\text{no tie}) = \frac{\binom{N-1}{(N-1)/2}}{2^{N-1}}. } P(no tie)=2N−1((N−1)/2N−1​)​.​

Closing Thoughts

The next time you see a “probability of tie” or “stay below” problem, think of it as a path trapped inside a polygon — and decide which of these three lenses brings the boundary into focus.

Thanks to Bertrand, André, and Catalan for showing that even the humblest ballot box hides deep geometry.
