Reading Notes for Pattern Recognizing and Machine Learning by Chris Bishop

Posted on 2024-04-04 Word count in article: 3.8k Reading time ≈ 3 mins.

This book Pattern Recognizing and Machine Learning by Christopher M. Bishop(2006) was recommended by my professor. Here are the reading notes.

Introduction

Basic Concepts

  1. The ability for the machine leanring algorithm to categorize new examples outside the training set are called generalization, which is the main goal in machine learning.

    • An exmaple of the preprocessing stage is when a machine learning algorithm is used to recognize handwritten digits.

    The images of the digits are typically translated and scaled so that each digit is contained within a box of a fixed size... This pre-processing stage is sometimes also called feature extraction.

  2. reinforcement learning is trying to maximize a reward in a given situation. However, reward has to be attributed appropriately to all the moves that led to it, which is a credit assignment problem. THe general feature of reinforcement learning is thr trade-off between exploration(machine tries out new kinds of actions to see how effective they are) and exploitation (make use of actions that are known to yield a high reward).   3. Root-Mean-Square(RMS) error: used to evaluate E(w) for the test data set. where \[E(w) = \frac{1}{2} \sum_{n=1}^{N}{(y(x_n, w) - t_n)^2}\]

and RMS is calculated:\[E_{RMS} = \sqrt{2E(w) / N}\]. The square root ensures that \(E_{RMS}\) is measured on the same scale and units as the target vaiable t ([U] Why do we ensure that, also where does the 2 come from)

Since \(E(w)\) is a quadratic function, its derivative would be a linear equation. There should be a one and only w* set where E(w) reaches minimum. We would use w* as our model.

  1. Problem of Overfitting. Suppose we train a model in this way: We use the function \(y = sin(2\pi{x})\), and we generate a training set by adding random noise to the target value. We now train the model with different polynomial functions of \(y = w_0 + w_1 * x^1 + ... + w_M * x^M\), where M denotes the order of the polynomial. We use 3 differnt models with different Ms: M = 0, M = 3, and M = 9. With a training set of 10 data points, we surprisingly find that: M = 0 fits the function \(y = sin(2\pi{x})\) badly, M = 3 fits the function most, M = 9 goes through every point in the data set but it fits the original function badly. We call M = 9 an overfitting.

    • This is quite resonable since for M = 9, there are \(w_0, w_1...w_{10}\) 10 degrees of freedom, so it would fit the training set perfectly. However, if we use RMS error to calculate the performance of different Ms models, we find that M = 9 behaves badly on new datas.

    • This feels strange because (1) M = 3 could be considered a special case for M = 9, so M = 9 should behave at least as good as M = 3. (2) consider the function \(y = sin(2\pi{x})\), it could be represented as a power series, so it was supposed to improve monotonically as we increase M.

    • If we check the w* for each model, we can see that as M increases, w* increases. So we observe oscillation in the polynomial of M = 9. Intuitively, polynomials with larger values of M are increasingly tuned to the random noises.

    • We can see that over-fitting problem becomes less severe as the size of the data set increases. It is said that the number of data points should be no less than some multiple(5/10) to the number of adaptive parmeters in the model. ([u] chapter 3 this changes? [U] over=fitting problem can be understood as "general property of maximum likelihood" ?)

    • This kind of doesn't make sense because we want the number of the parameters in a model(complexity of a model) to be influenced by the complexity of the problem, rather than the number of data points that we have.[U] this can be avoided by the Bayesian model.

    • Without the above approach, one technique to solve the problem is to add an addtional penalty term. \[E(w) = \frac{1}{2} \sum_{n=1}^{N}{(y(x_n, w) - t_n)^2 + \lambda{} ||w||^2}\], where \[||w||^2 = w_1^2 + w_2^2+....w_M^2\]. the \(\lambda\) term determines the importance of the two terms. Do note that \(w_0\) is omitted because it causes the results to depend on the choice of origin for the target variable. ([U] or it may be included. Discussed in section 5.5.1). This is called a shrinkage method. The case of quadrati regularizer is called ridge regression, in the context of neural networks, it is called weight decay.

    • Determining the complexity for the model is important. The result above shows a way of seperating two different sets, which is the training data set used to determine w, and a validation set or a hold-out set used to optimize the model complexity.
