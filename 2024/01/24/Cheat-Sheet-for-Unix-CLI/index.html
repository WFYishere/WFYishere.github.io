Cheat Sheet for Unix CLI

Posted on 2024-01-24 Edited on 2024-02-22 Word count in article: 3k Reading time ≈ 3 mins.

Learnt Unix in NUS CS2030S. Here's a cheatsheet.

Original Text Here

Background

  • Ctrl + D : end of input/ exit shell

  • Ctrl + Z : suspend the current running program. This pauses the execution of the program (but not terminates it). Resume executing in the background with the command bg Back to execution in the foreground again with the command fg.

  • Ctrl + C : terminate the current running program.

  • Ctrl + S : freeze the terminal. Ctrl + Q : resume the terminal.

  • Ctrl + P & Ctrl + N: See the previously executed commands.

  • Tab : complete the name of a command/file

  • man : look at the manuals. e.g. : man ls

Essentials

Directory

Symbols

  • ~ : The home directory. ~bob means the home directory of a user named bob.

  • . : Current working directory.

  • /: Root directory.

  • .. : Parent directory.

Path

  • Absolute Path: /home/o/ooiwt

  • Relative Path: ../o/ooiwt

  • pwd : Print current working directory.

  • ls : Check the list of contents. Contents that starts with . are hidden, use ls -a to check all contents. ls -F to see more information. ls -l to see information including the size and date.

  • mkdir : make a directory

  • cd : brings you to the home directory

Files

  • rmdir : remove a subdirectory, it must be empty.

  • cp :

    cp -r copys the whole directory: $ cp -r ../workshop .

* If there is an existing file with the same name, cp will overwrite the existing file without warning.

  • mv : Rename a content.

    When the destination of mv is a directory, then it moves the source to the destisnation

    It's better to use mv -i as it asks if you want to overwrite a file

  • rm : removes the file.

    *It's better to use rm -i.

  • cat : Take a look at the content of a file.

    less : look at less content of a file.

      In less, use `space` to move forward,  `b` to move back, `q` to quit.

File Permission

What:

  • r : read - 4

  • w : write - 2

  • x : executing a file / access contents and info of the directory - 1

Permission are used in two ways:

  1. rwx or r-x etc.

  2. Numerical notation like : 5 = r-x

Who

u is the user who owns the file; g refers to the users in the same group as the user; and o are all the other users.

For instance, the permission of 644, or rw-r--r--, on a file means that:

- the owner can read and write

- the group users can only read

- all the other users can only read

Check and Change of Permission

Check:

$ ls -l test.txt
-rw-r--r--@ 1 ooiwt  staff  64 Jul 28 09:52 test.txt

Change:

$ chmod 666 test.txt
$ ls -l test.txt
-rw-rw-rw-@ 1 ooiwt  staff  64 Jul 28 09:52 test.txt

Or:

$ chmod o-w test.txt
$ ls -l test.txt
-rw-rw-r--@ 1 ooiwt  staff  64 Jul 28 09:52 test.txt

Eliminates the writing permission of a user.(use o+w to give the permission)
