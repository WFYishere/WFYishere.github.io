[LeetCode] 49 - Group Anagrams

Posted on 2024-06-01 In Solutions , LeetCode , Medium Word count in article: 3k Reading time ≈ 3 mins.

Interesting question... How could I have not thought of hashMap

Problem Description

Given an array of strings strs, group the anagrams together. You can return the answer in any order.

An Anagram is a word or phrase formed by rearranging the letters of a different word or phrase, typically using all the original letters exactly once.

\(1 <= strs.length <= 10^4\)

\(0 <= strs[i].length <= 100\)

Solution 1 HashMap

Runtime Analysis

Runtime: O(n), Memory: O(n), where n is the number of strings

Runtime: 7ms, Memory: 47.51MB

Idea & Code

Idea: We use HashMap to store the anagrams and use the string itself as key of the hashMap. This greatly imporves runtime.

Solution 2 BruteForce

Runtime Analysis

Runtime: O(n^2), Memory: O(n), where n is the number of strings

Runtime: 890ms, Memory: 47.02MB

Idea & Code

Idea: Bruteforce the question.

Conclusion

How could I have not thought of HASHMAP!

I think that we should think about using hashMap whenever we are using a data structure that requires QUICKFINDING! E.g. here we need quickfind of Anagrams.

Or maybe we should think of hashMap whenever we need to iterate through a whole array.

Also, here we can see that the length of the string is at most 100. Thus, we can use sorting algorithm on the strings.
