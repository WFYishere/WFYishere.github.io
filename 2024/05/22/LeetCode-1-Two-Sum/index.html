    [LeetCode] 1 - Two Sum

    Posted on 2024-05-22 Edited on 2024-05-23 In Solutions , LeetCode , Easy Word count in article: 3.7k Reading time ≈ 3 mins.

    Easy problem but various solutions.

    Problem Description

    Find the problem description here

    Given an array of integers nums and an integer target, return indices of the two numbers such that they add up to target.

    Each input would have exactly one solution, and you may not use the same element twice.

    \(2 <= nums.length <= 10^4\)

    \(-10^9 <= nums[i] <= 10^9\)

    \(-10^9 <= target <= 10^9\)

    Solutions (with code)

    BruteForce

    Runtime Analysis

    Runtime: O(n^2), Memory: O(n)

    Runtime: 45ms, Memory: 44.89MB

    Idea & Code

    use two for loops to search through all two numbers whose sum might match the target

    Sorting + Low-High Pointer

    Runtime Analysis

    Runtime: O(nlogn), Memory: O(n)

    Runtime: 9ms, Moemry: 44.60MB

    Analysis: Sorting: O(n log n), two pointers: O(n)

    Idea & Code

    1. Sort the array. To compensate for the information lost during the search, that is, the index of the array, we use pair to store the array

    2. Keep two pointers low and high. Initially, the two pointers points to the smallest and largest numbers in the array. For each time, we check if the sum of the two numbers that the pointers are pointing two is the target that we need. If the sum is smaller, we move the low pointer rightwards, if the sum is larger, we move the high pointer leftwards.

    Suggested Solutions

    Sorting + QuickFind

    Runtime Analysis

    Runtime: O(n log n), Space complexity: O(n)

    Analysis: Sorting: O(n log n), QuickFind: O(log n) for each time, run n times, thus O(n log n).

    Idea

    1. Sort the array.

    2. For each number in the array, we use QuickFind to find the target - number

    HashMap

    Please check here for detailed code.

    Runtime Analysis

    Runtime: O(n), Space Complexity: O(1) (\(10^9\))

    Runtime: 5ms, Space Complexity: 44.55MB

    Idea

    Use hashmap to store each components in the array, and try to find target - number in hashmap

    Optimization: We don't preprocess hashmap then search for the complement of the current number. Instead, we iterate through the array only once and push current number into the hashmap after checking whether the complement of current is in the hashmap.

    Note: Do note that here, we store nums[i] as key and i as value, since we search using nums[i]

    Conclusion

    This is an interesting problem, get to know more about arrays and hashmaps.
