[Leetcode] 15 - 3Sum

Posted on 2024-05-28 Edited on 2024-05-31 In Solutions , LeetCode , Medium Word count in article: 2.9k Reading time ≈ 3 mins.

An extended version of Two-Sum

Problem Description

Given an integer array nums, return all the triplets [nums[i], nums[j], nums[k]] such that i != j, i != k, and j != k, and nums[i] + nums[j] + nums[k] == 0.

Notice that the solution set must not contain duplicate triplets.

\(3 <= nums.length <= 3000\)

\(-10^5 <= nums[i] <= 10^5\)

Solution

Solution 1: hashMap Solution

Runtime Analysis

Runtime: O(n^2), Memory: O(n^2)

Runtime: 508ms, Memory: 51.6MB

Idea & Code

Idea:

  1. Sort the array

  2. Iterate throught the array and find the Two-Sum of -nums[i].

Solution 2: Low-High Pointer

Runtime Analysis

Runtime: O(n^2), Memory: O(n)

Runtime: 31ms, Memory: 51.43MB

Idea & Code

The Low-High Pointer approach would be a better solution. That is because, though both Low-High Pointer and HashMap takes O(n) time, we need extra space for HashMap solution to make sure that the array includes no duplicates.

Conclusion

Interesting question. Made me realize that I should consider the complexity(in terms of time and difficulty) before choosing an approach. E.g. Low-High Pointer is significantly easier approach here.
