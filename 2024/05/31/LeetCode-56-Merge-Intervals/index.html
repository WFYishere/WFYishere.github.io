[LeetCode] 56 - Merge Intervals

Posted on 2024-05-31 In Solutions , LeetCode , Medium Word count in article: 2.1k Reading time ≈ 2 mins.

Easy problem, helps me understand more about sorting and arrays in java.

Problem Description

Given an array of intervals where intervals[i] = [starti, endi], merge all overlapping intervals, and return an array of the non-overlapping intervals that cover all the intervals in the input.

\(1 <= intervals.length <= 10^4\)

\(0 <= starti <= endi <= 10^4\)

Solution

Runtime Analysis

Runtime: O(n log n), Memory: O(n)

Runtime: 7ms, Memory: 46.53MB

Idea & Code

Idea: The main problem that we need to solve here is that we need to sort the intervals in ascending order of the starting points. Here we use a simple comparator.

Once the intervals are sorted, we just need to intuitively merge the intervals one by one.

Conclusion

At first I was thinking of divide and conquer and maybe we can use an idea similar to that of merge sort. We first "merge" the two parts of the array(that is, merge all overlapping intervals), then try to merge the two "merged" parts. However, when we try the 2nd step(merging the two parts), we would need the two parts to be both sorted. At that time I finally realize that I have made the question too complicated and that we just need to sort the intervals and intuitively merge them.
