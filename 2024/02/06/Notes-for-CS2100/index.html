Number System

Introduction

Decimal Number System 1. Weighted-posistional number system 2. Base(radix) is 10 3. Each position has a weight of power of 10. (1.2 where 2 has a weight of \(2^{-1}\)) 4. e.g.: 7594.36 is a 6 digit number with 2 decimal places

Prefix 1. 0 for octal(8) 2. 0x for hexadecimal

Converting Decimals

American Standard Code for Information Interchange

  • 7 bits, plus 1 parity bit(odd or even parity) (Count the number of 1 bits and check whether it is even/odd(_0101101: odd parity should be 0))

  • Converting: '9' - '0' = 9

Signed and Unsigned Values

Ways to represent

  1. Sign-and-Magnitude

    • sign bit: 0 for + and 1 for -
    • \({10100000}_{sm} = 33_{10}\)

  2. 1s complment

    • -x = \(2^{n}\) - x - 1
    • to negate x, just invert all the bits
    • MSB still represents the sign, 0 for positive, 1 for negative
    • MSB's weight: (in n bits, ) \(-2^{n-1} + 1\)

  3. 2s-complement

    • -x = \(2^n\) - x
    • Negate x: invert all bits and add 1
    • shortcut: from right to left, copy everything until the first 1. invert everything to the left of the first 1. (e.g: 00101000 -> 11011000)
    • MSB still represent the sign, 0 for positive
    • in a 4-bit 2s number, the MSB has a weight of -8 ([U] Prove that?)

  4. Excess Notation

    • Allows the range of values to be split evenly betweent the positive and negative values.
    • the Excess-n notation means that in order for a negative number x to be negated, you need to add n to it. (e.g.: for the excess-4 notation, add 4 to -2 inorder to get 2)

For a 4-bit system, - SM: 0 to 7, -0 to -7 - 1s: 0 to 7, -0 to -7 - 2s: 0 to 7, -1 to -8 -8 is 1000

So

Complement on Fractions For fractions, simply ignore the '.'

Representation of Numbers

Fixed vs Floating

  1. Fixed Point Representation The allocated number for the number part and fractional part are fixed.

  2. Floating Point Representation Allows representation of very large and very small numbers.

IEEE 754 Floating-Point Rep.

  1. Components:sign - exponent - mantissa(fraction)

The sign is for the mantissa, not the exponent

  1. Format: Single-precision(32 bits)

    • 1-bit sign, 8-bit exponent with bias127(excess-127), 23-bit mantissa

    • The Mantissa is normalised with an implicit leading bit 1.

      This means that we will always keep the part to the left of the point 1. Only the left part will be stored. e.g.: (110.1\()_2\) = 1.10\(1_2\) * \(2^2\), only 101 is stored in the mantissa. Normalisation: 123.2 \(10^{0}\) = 1.232 * \(10^{2}\)*

  2. E.g.: To find the IEEE 754 representation of -6.5 \[-6.5_{10} = -110.1_{2} = -1.101_{2} * 2^{2}\] Since \[Exponent = 2 + 127 = 129 = 10000001_2\] \[ (-110.1)_2 = 1_{sign}10000001_{exponent}{101 0000 ... 0000}_{mantissa}\] Or in hexadecimal \[1 10000001 1010000...00000_2 = C0D00000_{16}\]

ISA

MIPS((Microprocessor without Interlocked Pipelined Stages)

Introduction

  1. How the computer work:

    • programming: High-level language(C++/java) - Compiler -> Assembly Language - Assembler -> Machine Language Instructions(1000 1100...) -> MIPS processor

    • components: Processor & Memory.

      • Bus: the data connector between the Processor and Memory.

      • Processor: Fast. Contains ALU; Registers(temporary value storage)

        • Registers:

          • A typical architecture has 16 to 32 registers.

          • Registers have no data types(assumes that the data type is correct).

          • Referered by number($0)/name($a0)

          • $at(register 1) is reserved for the assembler, $k0-k1(26-27) reserved for the operating system

      • Memory: Instruction and data are stored in memory.

  2. ISA: abstraction between the hardware and low-level software(set of instructions) - manual for the hardware.

  3. What's the use?

    • Include everything programmers need to know to make the machine code work correctly.

    • allow CPU designers to design functionalities in different hardware

  4. Assembly Code Types:

    • Memory Instruction: move data into registers. r0 <- load i ; move data from register into memory: store

    • Reg-to-Reg Arithmetic: Arithmetic operations can work directly on registers only. It can also take in constant values(instead of from registers).

    • Control Flow: Instructions to change the control flow based on conditions(e.g. Repetition(loop) / Seection(if-else) ).

More Instructions

Memory Organisation

  1. Given a k-bit address, the address space is of size \(2^k\), which is 8 locations. The memory might contain 1 byte of data in every location/address(byte addressing).

  2. Transfer Unit:

    • Using distinct memory address, we can access a single byte(byte addressable) / word (word accessable). * Word is Usually \(2^n\) bytes and is the common unit of transfer between processor and memory.

    • Word Alignment: Words are aligned in memory if they begin at a byte address that is a multiple number of bytes in a word. (e.g. If 4-byte word, then 0-3, 4-7 aligned; 1-4, 5-9 not aligned)

  3. MIPS is a load-store register architecture

    • 32 registers, each 32-bit long

    • Memory address are 32-bit long(\(2^{32}\) addresses)

    • Each word contains 32 bits (4 bytes) (\(2^{30}\) memory words)

Memory Instruction

  1. Load Word: lw $t0, 4($s0), the memory word at Mem[$s0 + 4] is loaded into $t0 4 is the offset and should be multiple of 4(length of memory)

  2. Store Word: sw $t0, 12($s0), value at $t0 is stored into Mem[$s0 + 12]

  3. Load and Store Arrays: $s3 contains the base address(A[0]), h is mapped to $s2

  1. lb and sb used to load byte and store byte (when $t2 is 32 bits long, we use the lowest byte.)

  2. ulw and uls are pseudo-Instructions for unaligned load word

Consecutive word address in machines with byte-addressing do not differ by 1, it differs by length of word!

Instruction Formats

MIPS Instruction Encoding

Each MIPS Instruction has a fixed length of 32 bits(word alligned).

Remember that the opcode and funct values on the sheet are hexadecimal!

MIPS Reference Data

  1. R format(Register): op $r1, $r2, $r3 (e.g.: srl sll add sub)

    • opcode rs rt rd shamt funct

    • only 32 registers, so only 5 bits for each registers. Same for shamt(), since shifting 32 bits clea the entire register. opcode and funct are 6 bits.

    • opcode are 0 for all R-Format.

    • funct specifies the instruction

    • rs: source register; rt: target register; rd: Destination register;

    • shamt: amount a shift instruction will shift by. 0 for all non-shift instructions

  2. I format(Immediate): op $r1, $r2, Immd (e.g.: lw sw beq bne)

    • opcode rs rt immediate

    • opcode (6 bits): uniquely specifies an instruction

    • rs rt (5 bits): rs: source register. rt: destination Note the difference in I and R format!

    • immediate(16 bits) : signed integer(2s complement).

    • For beq/bne, immediate is not enough to specify the entire target address(memory address: 32 bits). Thus, we do PC-Relative Addressing (a branch often change the PC value by a small amount)! When branch is not taken, PC = PC + 4, when taken, PC = (PC + 4) + immdiate * 4(1 word = 4 address). Where immediate is a signed two's complement integer (the number of instruction needed to jump).

    • Program Counter: a special register that keeps address of instruction to be executed in the processor.

  3. J format(Jump): op Immd

    • opcode target-address

    • opcode (6 bits)

    • target-addres (26 bits): Assume we go word by word, then it's 28 bits. Where do we get the other 4 bits? We choose the 4 most significant bits from PC + 4. Thus the maximum jump range would be \(2^28\) bytes = 256 MB. Effective Address: target-address + 00(by word)

    • Steps: write out address(in binary) of PC + 4, write out address to jump to. Take the middle 26 bits and store into target address(despite the 4 significant bits and the last 2 0s)

Combinatory Circuit

Concept

We need a control unit to generate the signals for each multiplexer/Unite based on op-code and possibly function code. A combinatory circuit is a machine that changes the output immediately as the input changes. A Sequential Circuit is a machine that changes output only depending on a clock signal and the input(e.g. PC).

ALU

Introduction to ALU

ALU is a combinatory circuit.

  • If Ainvert = 1, the multiplexer for A would choose A'.

  • Operation: Choose from the 3 results: and (00) / or(01) / add(10)

Do note that for subtraction, in order to get A - B from A + B', we set the Cin to be 1. (in 2s complement)

  • We use A' to represent not A, use A.B to represent A and B, use A+B to represent A or B

Boolean Functions

Basics

Literals: boolean variables on its own or complemented form

SOP and POS

Product Term: the and of several literals or a single literal. E.g. x.y'.z

Sum Term: The or of several literals

Sum Of Product: the sum of several product terms or a single product term.

Product Of Sum: the prodcut of several sum terms or a single sum term.

Every boolean expression can be written as SOP / POS term.

Min and Max

Min Term: a min term of n variables is a product term of n literals.

  • The min and max term of 2 variables is named m0-m3:

    • m0: x'.y' / x+y

    • m1: x'.y / x+y'

    • m2: x.y' / x' + y

    • m3: x.y / x' + y'

mx' = Mx

A good way to memorize: imagine x' as 0 and x as 1. Then x'y' is 00, which is 00, x'y is 01, which is 1

Max Term: a max term of n variables is a sum term of n literals.

Canonical and Normal

Sum of min terms = Canonical SOP, we write this by looking at places where the output is 1.

Product of max terms = Canonical POS, we write this by looking at places where the output is 0.([U] check this out yourself!)

Converting Canonical SOP and POS: \[m0 + m1 + m5 = \sum{m(0,1,5)} = \prod{M(2,3,4,6,7)}\]

  1. Pipeline rate is limited by slowest pipeline stage and dependencies

    • The five execution stages in MIPS each takes 1 clock cycle.
- There will be additional registers between each stage. 

    - IF/IF: Register numbers for reading two registers; 16-bit offset to be sign-extendeed to 32-bit; PC + 4

    - ID/EX: Data Values from Register File; 32-bit immediate value; PC+4

    - EX/MEM: (PC+4+Immediate * 4); ALU result; isZero?; Data Read 2 from register

    - MEM/WB: ALU result; Mem Read data

- However, because of the pipeline design, the Write Register Number might be covered by the followed instructions. Thus, we need the Number to be carried through the five stages.

- Control Signals are generated in the decode stage. Thus, the control signals will be carried through the EX/MEM/WB stages(However, the signals for previous stages will not be carried once used.That is, signals for EX stage will not be carried in the EX/MEM register file.)

Performance:

  • Single-Cycle Datapath: Each instruction takes one cycle \[CT_{single} = max_{max(\sum{T_k})}\]

\[Time_{swq} = I * CT_{seq}\]

  • Multi-Cycle Processor

\[\text{Cycle Time: }CT_{multi} = max(T_k\text{(time for each stage)})\]

\[Time_{multi} = I * AverageCPI\text{(average clock per instruction)} * CT_{multi}\]

  • Pipelining Processor:

\[\text{Cycle Time: }CT_{pipeline} = max(T_k) + T_d\text{(time for accessing pipeline register)}\]

\[\text{Cycles needed for I instructions: }I + N -1 \text{N-1 cycles needed to fill up the pipeline}\]

\[\text{Time: } Time_{pipeline} = (I + N - 1 ) * CTpipeline\]

For ideal cases, where each stage takesthe sae amount of time, and there is no pipeline overhead(Td = 0), I >> N. \[Speedup_{pipeline} = \frac{Time_{No Pipeline}}{Time_{With Pipeline}} = N\]

  • Notes: \[1ns = 1 * 10^{-9} s \]

Hazards

  1. Structural Hazards

    • A hardware is used by two differet instructions!

      • Memory: split into instruction memory and data memory

      • Register: Split the stage into two halfs, where we write the register in the first half and read in the second half(We cannot swap this becuase the instructions are always based on previous instructions)

  2. Data Dependency

    • Read-After-Write Dependency: The only depndency that causes a pipeline hazard. Lets say data is written into register $2 instruct1, and it is read in instruction 2.

      • Solution: Forwarding: In instruction 2 execution stage, we override the data that was supposed to be read(that is, the original data before instruct1 was executed), and override it with the new data that is generated in the execution stage of instruct 1. This means that the data is passed to instruct 2 even before the write data stage of intruct1. The data is passed to the next instruction immmediately after execution.

      • Solution: Stall! What if data is needed before executed!

      • With only stall, we can only execute IF instruction as before, and ID instruction would be executed after the data is generated. With stall and forwarding, we can execute IF and ID as before, and we override data from ID technique in EX stage. That is, only EX stage will be delayed

  3. Control Dependency

    • Early Branch: We decide whether to branch in the instruction decode stage. This reduce the 3 clock waiting time to 1. However, this brings problem of data dependency since the data is needed in the ID stage, thus further stall is still needed.

      • If nothing, then 1 cc needed. If one add instruct before, then 2cc, if lw, then 3cc needed

    • Branch Prediction

      • Assume that branch is not taken. If prediction is wrong, we flush all the wronginstructions.

      • We combine this technique with the early branch technique to speed up(3cc delay -> 1cc delay).

      • This technique saves one delay everytime a correct prediction is made, in comparison to Early Branching

    • Delayed Branch:

      • Observation: Branch outcomes takes X number of Cycles to be know would cause X delay.

      • Move non-control dependent instructions into the X slots following a branch. This is done by the compiler. If no such instruction is find, then a no-op(NOP) instruction is put into the delay slot.
