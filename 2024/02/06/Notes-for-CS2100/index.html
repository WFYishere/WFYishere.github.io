Number System

Introduction

Decimal Number System 1. Weighted-posistional number system 2. Base(radix) is 10 3. Each position has a weight of power of 10. (1.2 where 2 has a weight of \(2^{-1}\)) 4. e.g.: 7594.36 is a 6 digit number with 2 decimal places

Prefix 1. 0 for octal(8) 2. 0x for hexadecimal

Converting Decimals

American Standard Code for Information Interchange

  • 7 bits, plus 1 parity bit(odd or even parity) (Count the number of 1 bits and check whether it is even/odd(_0101101: odd parity should be 0))

  • Converting: '9' - '0' = 9

Signed and Unsigned Values

Ways to represent

  1. Sign-and-Magnitude

    • sign bit: 0 for + and 1 for -
    • \({10100000}_{sm} = 33_{10}\)

  2. 1s complment

    • -x = \(2^{n}\) - x - 1
    • to negate x, just invert all the bits
    • MSB still represents the sign, 0 for positive, 1 for negative
    • MSB's weight: (in n bits, ) \(-2^{n-1} + 1\)

  3. 2s-complement

    • -x = \(2^n\) - x
    • Negate x: invert all bits and add 1
    • shortcut: from right to left, copy everything until the first 1. invert everything to the left of the first 1. (e.g: 00101000 -> 11011000)
    • MSB still represent the sign, 0 for positive
    • in a 4-bit 2s number, the MSB has a weight of -8 ([U] Prove that?)

  4. Excess Notation

    • Allows the range of values to be split evenly betweent the positive and negative values.
    • the Excess-n notation means that in order for a negative number x to be negated, you need to add n to it. (e.g.: for the excess-4 notation, add 4 to -2 inorder to get 2)

For a 4-bit system, - SM: 0 to 7, -0 to -7 - 1s: 0 to 7, -0 to -7 - 2s: 0 to 7, -1 to -8 -8 is 1000

So

Complement on Fractions For fractions, simply ignore the '.'

Addition and Subtraction

2s Complement

Simply disgard the additional bit U

For subtraction, add \((-x)_{2s}\) U

Overflow Checking

If the carry in and carry out of the MSB is different, then a overflow occurs.

e.g.: 1001 + 1101 - in:0 out: 1 thus an overflow

1s Complement

  1. Addition: If there is a carry out of the MSB, add 1 to the result ([U] why does this work?)
  2. Subtraction: add \((-x)_{1s}\)

Representation of Numbers

Fixed vs Floating

  1. Fixed Point Representation The allocated number for the number part and fractional part are fixed.

  2. Floating Point Representation Allows representation of very large and very small numbers.

IEEE 754 Floating-Point Rep.

  1. Components:sign - exponent - mantissa(fraction)

The sign is for the mantissa, not the exponent

  1. Format: Single-precision(32 bits)

    • 1-bit sign, 8-bit exponent with bias127(excess-127), 23-bit mantissa

    • The Mantissa is normalised with an implicit leading bit 1.

      This means that we will always keep the part to the left of the point 1. Only the left part will be stored. e.g.: (110.1\()_2\) = 1.10\(1_2\) * \(2^2\), only 101 is stored in the mantissa. Normalisation: 123.2 \(10^{0}\) = 1.232 * \(10^{2}\)*

  2. E.g.: To find the IEEE 754 representation of -6.5 \[-6.5_{10} = -110.1_{2} = -1.101_{2} * 2^{2}\] Since \[Exponent = 2 + 127 = 129 = 10000001_2\] \[ (-110.1)_2 = 1_{sign}10000001_{exponent}{101 0000 ... 0000}_{mantissa}\] Or in hexadecimal \[1 10000001 1010000...00000_2 = C0D00000_{16}\]
